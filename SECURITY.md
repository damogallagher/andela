# Security Policy

## Supported Versions

This challenge project is maintained on the `dev` branch until final submission. Security fixes should target `dev`.

## Reporting A Vulnerability

Do not open a public issue for a suspected vulnerability. Use a GitHub private security advisory for `damogallagher/andela` when available, or contact the repository owner directly.

Please include:

- Affected route, command, workflow, or file.
- Steps to reproduce.
- Expected and observed impact.
- Any safe proof-of-concept payloads or logs.

The project is local-first and should not be used to scan production infrastructure without a separate security review.
